# 第 20 章 深度学习基础

这一章把神经网络继续往深度学习方向展开。前面已经学过神经元、前向传播、反向传播和正则化，这里重点整理训练深度网络时最常见的组件。

## 20.1 深度学习解决什么问题

传统机器学习通常需要人工设计特征。例如房价预测里手动选择面积、房间数、房龄；垃圾邮件分类里手动设计词频、关键词等。

深度学习更强调表示学习：模型从数据中逐层学习特征。浅层特征通常更接近原始数据，深层特征更抽象。

| 数据类型 | 常见模型 | 学到的表示 |
| --- | --- | --- |
| 图像 | CNN、ViT | 边缘、纹理、部件、类别语义 |
| 文本 | RNN、Transformer | 词义、上下文、句子关系 |
| 图结构 | GNN | 节点、边和邻域结构 |
| 生成任务 | VAE、GAN、扩散模型 | 数据分布和生成过程 |

深度学习不是单独的一种任务，而是一类更强的建模方法。分类、回归、聚类、推荐、生成都可以用深度学习方法实现。

## 20.2 激活函数

激活函数负责引入非线性。如果没有激活函数，多层线性层叠加后仍然等价于一个线性模型。

常见激活函数：

| 激活函数 | 形式或特点 | 常见用途 |
| --- | --- | --- |
| Sigmoid | 输出在 `0` 到 `1` | 二分类输出层 |
| Tanh | 输出在 `-1` 到 `1` | 早期 RNN |
| ReLU | $\max(0, x)$ | 隐藏层常用 |
| Leaky ReLU | 负半轴保留小斜率 | 缓解 ReLU 死亡问题 |
| GELU | 平滑的非线性 | Transformer 常见 |

ReLU 计算简单，正半轴梯度稳定，因此在深度网络里非常常用。但如果学习率过大，某些神经元可能长期输出 `0`，这叫 ReLU 死亡。

## 20.3 参数初始化

神经网络不能把所有权重初始化为相同值，否则同一层的神经元会学到一样的特征，无法打破对称性。

常见初始化目标是：让前向传播时激活值不要越来越大或越来越小，也让反向传播时梯度保持合适尺度。

| 初始化 | 常见搭配 |
| --- | --- |
| Xavier / Glorot | Sigmoid、Tanh |
| He 初始化 | ReLU、Leaky ReLU |

偏置通常可以初始化为 `0`，因为权重已经通过随机初始化打破了对称性。

## 20.4 归一化

归一化可以让训练更稳定。

| 方法 | 归一化范围 | 常见场景 |
| --- | --- | --- |
| Batch Normalization | 一个 batch 内同一特征 | CNN、前馈网络 |
| Layer Normalization | 单个样本内部特征 | RNN、Transformer |

BatchNorm 的主要作用是稳定训练过程、加速收敛。它依赖 batch 统计量，因此 batch size 很小时可能不稳定。

LayerNorm 不依赖 batch 大小，训练和推理阶段行为更一致，所以在 Transformer 中非常常见。

## 20.5 Dropout 和权重衰减

Dropout 在训练时随机丢弃一部分神经元，让模型不要过度依赖某些局部组合。测试时使用完整网络。

权重衰减通过惩罚过大的权重缓解过拟合。它和 L2 正则化关系很近，都是让模型参数不要变得过大。

| 方法 | 作用 |
| --- | --- |
| Dropout | 减少神经元之间的共适应 |
| 权重衰减 | 限制权重大小 |
| 数据增强 | 扩充训练变化，提高鲁棒性 |
| 早停 | 验证集变差时停止训练 |

正则化不是为了让训练集误差最低，而是为了让模型在新数据上更可靠。

## 20.6 优化器

深度学习里常用小批量梯度下降。每次用一个 mini-batch 估计梯度，再更新参数。

常见优化器：

| 优化器 | 直观理解 |
| --- | --- |
| SGD | 按当前梯度更新 |
| Momentum | 使用历史梯度方向，减少震荡 |
| RMSProp | 根据梯度平方调整步长 |
| Adam | 结合 Momentum 和自适应学习率 |
| AdamW | Adam 加解耦权重衰减，训练大模型常用 |

Momentum 的作用是在相关方向上加速学习，同时抑制来回震荡。Adam 实践中很常用，但并不意味着不用调学习率。

## 20.7 学习率策略

学习率是最重要的超参数之一。

| 现象 | 可能原因 |
| --- | --- |
| 损失下降很慢 | 学习率太小 |
| 损失剧烈震荡 | 学习率偏大 |
| 损失变成异常值 | 学习率过大或梯度爆炸 |

常见学习率策略：

- 阶梯式下降。
- 指数衰减。
- 余弦退火。
- 学习率预热。

预热通常用于训练初期，让学习率从较小值逐渐升高，避免一开始更新过猛。

## 20.8 训练深度网络的基本流程

一个完整训练流程通常是：

```text
准备数据 -> 构建模型 -> 选择损失函数 -> 选择优化器 -> 训练 -> 验证 -> 调参 -> 测试
```

训练时要同时看训练集和验证集表现：

| 训练误差 | 验证误差 | 判断 |
| --- | --- | --- |
| 高 | 高 | 欠拟合 |
| 低 | 高 | 过拟合 |
| 低 | 低 | 拟合较好 |

调参时不要只看训练损失。真正关心的是模型能否泛化到没见过的数据。

## 20.9 前向传播、损失函数和反向传播

深度学习训练可以拆成三件事：

```text
前向传播：用当前参数得到预测
损失函数：衡量预测和真实标签的差距
反向传播：计算每个参数应该如何调整
```

以分类任务为例，输入 $x$ 经过多层网络得到 logits：

$$
z = f_\theta(x)
$$

logits 不是概率，通常还要经过 Softmax：

$$
p_k =
\frac{e^{z_k}}{\sum_{j=1}^{K} e^{z_j}}
$$

其中 $p_k$ 表示模型认为样本属于第 $k$ 类的概率。

多分类常用交叉熵损失：

$$
L = -\sum_{k=1}^{K} y_k \log p_k
$$

如果真实类别是第 $c$ 类，one-hot 标签里只有 $y_c = 1$，损失就等价于：

$$
L = -\log p_c
$$

也就是说，模型给真实类别的概率越低，损失越大。

反向传播不是一种新的优化算法，它只是高效计算梯度。真正决定参数怎么走的是优化器，例如 SGD、Adam。

## 20.10 logits、概率和标签

深度学习分类里容易混淆三个东西：

| 名称 | 含义 |
| --- | --- |
| logits | 模型最后一层的原始输出 |
| probability | logits 经过 Sigmoid 或 Softmax 后的概率 |
| label | 真实标签 |

二分类常见做法是输出一个 logit，再用 Sigmoid 得到正类概率。多分类常见做法是输出 $K$ 个 logits，再用 Softmax 得到 $K$ 个类别概率。

实际框架里，有些损失函数会把 Sigmoid 或 Softmax 和交叉熵合在一起计算。例如 PyTorch 的 `CrossEntropyLoss` 接收的是 logits，不需要手动先做 Softmax。这样数值更稳定。

## 20.11 梯度消失和梯度爆炸的根源

深层网络中，梯度要经过很多层链式相乘。如果每层导数都小于 `1`，连乘后会越来越小；如果每层导数都大于 `1`，连乘后可能越来越大。

这就是梯度消失和梯度爆炸的直观来源。

常见缓解方式：

| 方法 | 作用 |
| --- | --- |
| ReLU | 减少饱和区间带来的梯度消失 |
| 合理初始化 | 保持激活和梯度尺度稳定 |
| BatchNorm / LayerNorm | 稳定中间分布 |
| 残差连接 | 给梯度提供更短路径 |
| 梯度裁剪 | 限制过大的梯度 |

残差连接尤其重要。它让网络学习的是：

$$
H(x) = F(x) + x
$$

如果某一层暂时学不到有用变换，可以让 $F(x)$ 接近 `0`，至少不破坏输入信息。这让非常深的网络更容易训练。

## 20.12 训练时应该观察什么

训练深度网络时，不应该只看最后准确率。更重要的是观察训练过程。

| 观察对象 | 说明 |
| --- | --- |
| 训练损失 | 是否稳定下降 |
| 验证损失 | 是否开始上升 |
| 训练准确率 | 模型是否有能力拟合训练集 |
| 验证准确率 | 泛化能力是否提升 |
| 学习率 | 是否过大或过小 |
| 梯度范数 | 是否出现梯度爆炸 |

几个典型现象：

- 训练损失和验证损失都高：模型容量不足、训练不够、学习率不合适。
- 训练损失低但验证损失高：过拟合，需要正则化、数据增强或更多数据。
- 损失突然变成异常值：学习率过大、梯度爆炸、数据中有异常值。
- 训练很慢但方向正确：学习率可能偏小，或模型/数据管线效率低。

深度学习调参的基本顺序是：先让模型能在小数据上过拟合，再扩大到完整训练集，最后处理泛化问题。
