# 第 15 章 深度学习训练技巧

第 13 章和第 14 章已经说明了神经网络的表示方式和基本训练流程。真正训练深层网络时，还会遇到一批新的问题：激活函数选择、参数初始化、梯度消失和爆炸、归一化、正则化、优化器和学习率策略。

这些技巧不是零散的“调参经验”，它们都围绕同一个目标：让深层网络的信号在前向传播和反向传播中保持稳定，并让模型在训练集之外仍然有较好的泛化能力。

## 15.1 深层网络为什么难训练

浅层模型的参数少，计算路径短，梯度传播比较直接。深层网络有很多层，输出误差要通过多层函数一路传回前面层。每经过一层，梯度都会乘上这一层的导数和权重。

如果这些乘法因子长期小于 1，梯度会越来越小，前面层几乎学不动；如果长期大于 1，梯度会越来越大，训练可能震荡甚至数值溢出。

这就是梯度消失和梯度爆炸的根源。

## 15.2 激活函数

激活函数负责引入非线性。如果没有激活函数，多层线性变换叠加后仍然等价于一个线性模型。

早期神经网络常用 sigmoid：

$$
\sigma(x)=\frac{1}{1+e^{-x}}
$$

sigmoid 的输出在 0 到 1 之间，适合表示概率。但它在输入很大或很小时会饱和，导数接近 0，深层网络中容易导致梯度消失。

tanh 的输出在 -1 到 1 之间，比 sigmoid 更居中：

$$
\tanh(x)=\frac{e^x-e^{-x}}{e^x+e^{-x}}
$$

现代深度网络的隐藏层更常用 ReLU：

$$
ReLU(x)=\max(0,x)
$$

ReLU 在正半轴导数为 1，梯度传播更稳定，计算也简单。但 ReLU 在负半轴输出为 0，某些神经元可能长期不激活。Leaky ReLU 给负半轴保留一个很小斜率，用来缓解这个问题。

## 15.3 参数初始化

如果所有权重初始化为 0，同一层的神经元会得到相同梯度，训练后仍然相同。这样多个神经元不会学到不同特征。

随机初始化可以打破对称性，但随机值不能太大，也不能太小。太大容易导致激活值或梯度爆炸；太小则信号逐层衰减。

Xavier 初始化的思想是让每层输入和输出的方差尽量保持稳定，适合 tanh、sigmoid 等激活函数。He 初始化更适合 ReLU，因为 ReLU 会让一部分输出变为 0。

初始化的目的不是找到好答案，而是给优化过程一个稳定的起点。

## 15.4 梯度消失

梯度消失表现为：训练过程中后面层还能更新，前面层几乎不变。模型看似在训练，但底层表示没有学好。

sigmoid 和 tanh 在饱和区域导数接近 0，因此深层网络中更容易出现这个问题。RNN 在长序列上也容易梯度消失，因为误差需要沿时间步反复传播。

缓解方法包括：

- 使用 ReLU 或其变体。
- 使用合适的参数初始化。
- 使用归一化层。
- 使用残差连接。
- 对 RNN 使用 LSTM 或 GRU。

## 15.5 梯度爆炸

梯度爆炸表现为损失剧烈震荡，参数更新过大，甚至出现 `NaN`。

常见处理方法是降低学习率、使用梯度裁剪或改进初始化。梯度裁剪会限制梯度范数，防止一次更新过大。

在序列模型中，梯度爆炸尤其常见，因为同一个权重矩阵会在多个时间步中反复参与计算。

## 15.6 批归一化

Batch Normalization 对一个 mini-batch 内的激活值做归一化，使其均值和方差更稳定。它通常能加速收敛，并让训练过程更稳定。

批归一化不是简单减少参数，也不是替代激活函数。它的主要作用是稳定每层输入分布。

训练阶段，BatchNorm 使用当前 batch 的均值和方差；推理阶段，使用训练过程中累计的移动平均统计量。因此 batch size 太小时，BatchNorm 的统计量可能不稳定。

## 15.7 层归一化

Layer Normalization 不依赖 batch 统计量，而是在单个样本内部对特征做归一化。

这使它更适合 RNN 和 Transformer。Transformer 中通常在注意力层和前馈网络附近使用 LayerNorm，再配合残差连接稳定深层训练。

BatchNorm 和 LayerNorm 的关键差别是归一化维度不同：BatchNorm 依赖一批样本的统计量，LayerNorm 依赖单个样本内部的统计量。

## 15.8 Dropout

Dropout 是一种正则化方法。训练时随机让一部分神经元暂时失效，迫使网络不要过度依赖某些固定神经元组合。

可以把 Dropout 看作每次训练一个变薄的子网络。测试时使用完整网络，并根据实现方式保持输出尺度一致。

Dropout 在全连接层中更常见。在卷积层中要谨慎使用，因为卷积结构本身已经通过局部连接和参数共享带来约束。

## 15.9 权重衰减

权重衰减通常对应 L2 正则化。它会惩罚过大的权重，让模型对输入微小变化不那么敏感。

在深度学习中，AdamW 把权重衰减从 Adam 的梯度自适应更新中解耦出来，因此比直接在 Adam 中加入 L2 项更清晰。

权重衰减不是为了让训练误差最低，而是为了提高泛化能力。

## 15.10 优化器

SGD 每次根据当前 mini-batch 的梯度更新参数。它简单、稳定，但在复杂损失曲面上可能收敛较慢。

Momentum 在 SGD 基础上加入历史方向，使参数在一致下降方向上加速，在震荡方向上相互抵消。

Adam 同时使用一阶矩估计和二阶矩估计，可以为不同参数自适应调整更新幅度。它在实践中很常用，但并不意味着永远比 SGD 泛化更好。

选择优化器时要看任务、模型和数据规模。优化器能影响训练速度和稳定性，但不能替代数据质量和合理模型设计。

## 15.11 学习率策略

学习率控制每次更新步长。训练初期可以使用较大学习率快速下降，训练后期需要较小学习率精细收敛。

常见策略包括阶梯下降、指数衰减、余弦退火和学习率预热。

学习率预热常用于训练大模型。训练刚开始时参数还不稳定，如果直接使用较大学习率，容易造成训练震荡。预热先使用较小学习率，再逐渐升到目标值。

## 15.12 训练时应该观察什么

训练深度网络时，不能只看最后准确率。更可靠的做法是同时观察：

- 训练损失。
- 验证损失。
- 训练和验证指标差距。
- 学习率变化。
- 梯度范数。
- 是否出现 `NaN`。

如果训练损失下降而验证损失上升，通常说明过拟合。如果训练损失本身不下降，要优先检查学习率、数据、标签、初始化和模型实现。

深度学习训练技巧很多，但判断顺序应该清楚：先确认数据和目标函数正确，再确认模型能在小数据上过拟合，最后再调优化器、正则化和学习率策略。
