# 第 25 章 机器学习工程

前面的章节讲模型和算法，本章讲如何把机器学习做成一个能用、能评估、能迭代的系统。它包含误差分析、评价指标、数据构建、类别偏斜、大规模训练、上线监控和分布偏移。

## 25.1 从问题定义开始

机器学习项目不要一开始就调模型。第一步应该明确问题：

- 输入是什么？
- 输出是什么？
- 标签怎么来？
- 用什么指标评价？
- 错误成本是否对称？
- 线上使用场景和训练数据是否一致？

如果问题定义错了，后面模型再复杂也没用。

## 25.2 误差分析

误差分析是查看模型错在哪里，而不是凭感觉调参。

例如垃圾邮件分类器错误很多，可以抽样 100 个错误样本，统计错误来源：

| 错误类型 | 占比 |
| --- | --- |
| 药品广告 | 30% |
| 钓鱼链接 | 20% |
| 正常营销邮件 | 15% |
| 其他 | 35% |

如果某类错误占比很高，就优先针对那类问题改进数据或特征。

## 25.3 Precision、Recall 和 F1

类别不平衡时，准确率可能很误导。例如异常检测中，99% 样本都是正常样本，模型全部预测正常也能有 99% 准确率，但完全没用。

精确率：

$$
Precision=\frac{TP}{TP+FP}
$$

召回率：

$$
Recall=\frac{TP}{TP+FN}
$$

F1：

$$
F1=\frac{2PR}{P+R}
$$

F1 综合了精确率和召回率。

## 25.4 Precision 和 Recall 的权衡

提高分类阈值，模型更谨慎，通常 precision 上升、recall 下降。

降低分类阈值，模型更激进，通常 recall 上升、precision 下降。

不同业务场景选择不同策略：

| 场景 | 更关注 |
| --- | --- |
| 疾病初筛 | Recall，尽量少漏诊 |
| 垃圾邮件拦截 | Precision，避免误杀正常邮件 |
| 搜索召回 | Recall，先多找候选 |
| 精排展示 | Precision，保证展示质量 |

## 25.5 数据比模型更重要

很多机器学习系统的问题不是模型结构不够先进，而是数据不干净。

需要检查：

- 标签是否准确。
- 训练集和测试集是否分布一致。
- 是否有重复样本。
- 是否发生数据泄漏。
- 类别是否严重不平衡。
- 线上输入是否和离线数据格式一致。

数据质量决定模型上限。

## 25.6 数据增强

数据增强用于在不改变标签语义的前提下扩充训练样本。

图像任务常见增强：

| 增强方式 | 提升鲁棒性 |
| --- | --- |
| 随机裁剪 | 目标位置变化 |
| 随机翻转 | 左右方向变化 |
| 颜色扰动 | 光照变化 |
| 随机旋转 | 拍摄角度变化 |

但增强必须符合任务语义。比如数字 `6` 旋转后可能变成 `9`，这种增强就不一定合理。

## 25.7 大规模训练

数据量很大时，全量梯度下降每次更新都要扫描全部数据，成本太高。实际常用小批量梯度下降。

```text
取一个 mini-batch -> 前向传播 -> 反向传播 -> 更新参数 -> 取下一个 mini-batch
```

大规模训练还会使用数据并行：多个 GPU 分别计算不同 batch 的梯度，再同步更新参数。

## 25.8 在线学习

在线学习适合数据持续到来的场景。模型可以不断接收新样本并更新。

典型例子：

- 广告点击率预测。
- 推荐系统。
- 实时风控。
- 搜索排序。

在线学习要特别注意反馈回路。模型推荐什么，用户就更可能看到什么；用户看到什么，又会影响后续训练数据。

## 25.9 分布偏移

训练数据和线上数据分布不一致，会导致模型效果下降。

常见偏移：

| 类型 | 含义 |
| --- | --- |
| 协变量偏移 | 输入分布变了 |
| 标签偏移 | 类别比例变了 |
| 概念偏移 | 输入和标签关系变了 |

上线后需要持续监控数据分布和模型指标。

## 25.10 从实验到上线

一个机器学习系统通常经历：

```text
定义问题 -> 收集数据 -> 训练模型 -> 离线评估 -> 小流量实验 -> 上线监控 -> 迭代
```

上线后不能只看离线测试集分数，还要看延迟、成本、稳定性、可解释性和用户反馈。

## 25.11 上限分析

如果系统由多个模块组成，例如 OCR：

```text
图像 -> 文本检测 -> 字符切分 -> 字符识别 -> 后处理
```

可以通过上限分析判断哪个模块最值得改。方法是：假设某个模块变成完美，看最终系统能提升多少。

如果某模块变完美也只能提升 1%，就不值得优先投入；如果能提升 20%，它就是关键瓶颈。
