# 第 17 章 序列模型

序列模型用于处理有顺序的数据，例如文本、语音、时间序列和用户行为序列。

## 17.1 序列数据的特点

普通前馈网络假设输入特征一次性给出，而序列数据有时间或位置顺序。

例子：

| 数据 | 序列含义 |
| --- | --- |
| 文本 | 词语顺序 |
| 语音 | 声音帧顺序 |
| 股票价格 | 时间顺序 |
| 用户点击 | 行为发生顺序 |

序列模型要捕捉前后元素之间的依赖关系。

## 17.2 RNN

循环神经网络在每个时间步读取一个输入，并维护一个隐藏状态。

隐藏状态可以看成模型对过去信息的记忆：

$$
h_t = f(W_{xh}x_t + W_{hh}h_{t-1} + b_h)
$$

输出可以由隐藏状态得到：

$$
y_t = g(W_{hy}h_t + b_y)
$$

RNN 的参数在不同时间步共享，因此可以处理不同长度的序列。

## 17.3 BPTT

RNN 的训练使用通过时间反向传播，也就是 BPTT。

BPTT 会把 RNN 沿时间展开成一个很深的网络，然后从后往前传播误差。

计算复杂度和时间步长度有关。序列越长，反向传播路径越长，训练越困难。

## 17.4 梯度消失和梯度爆炸

RNN 容易出现梯度消失和梯度爆炸。

| 问题 | 表现 | 处理 |
| --- | --- | --- |
| 梯度消失 | 难以学习长期依赖 | LSTM、GRU、残差结构 |
| 梯度爆炸 | 损失震荡或异常 | 梯度裁剪 |

梯度裁剪会限制梯度范数，避免一次更新过大。

## 17.5 LSTM

LSTM 通过门控结构控制信息保存和遗忘，缓解普通 RNN 难以学习长期依赖的问题。

LSTM 中有细胞状态和三个主要门：

| 门 | 作用 |
| --- | --- |
| 遗忘门 | 决定从细胞状态中丢弃哪些信息 |
| 输入门 | 决定写入哪些新信息 |
| 输出门 | 决定当前隐藏状态输出什么 |

考试里常问：遗忘门决定从细胞状态中丢弃哪些信息。

## 17.6 GRU

GRU 是另一种门控循环网络，结构比 LSTM 简单。

GRU 常见门：

- 更新门。
- 重置门。

它参数更少，训练更快，在很多任务上效果也不错。

## 17.7 序列任务类型

序列模型可以处理多种输入输出形式：

| 类型 | 例子 |
| --- | --- |
| 多对一 | 情感分类 |
| 一对多 | 图像生成描述 |
| 多对多等长 | 词性标注 |
| 多对多不等长 | 机器翻译 |

Transformer 出现后，很多 NLP 任务从 RNN/LSTM 转向注意力模型，但理解 RNN 仍然有助于理解序列建模的基本问题。

## 17.8 隐藏状态到底记住了什么

RNN 的隐藏状态 $h_t$ 可以看成“到当前时间步为止的摘要”。它不是把过去所有输入原封不动保存下来，而是用固定长度向量压缩历史信息。

这带来一个问题：当序列很长时，早期信息很容易被后续输入覆盖。

例如句子很长时，普通 RNN 可能记不住开头的主语，导致后面预测动词形式或指代关系时出错。

LSTM 和 GRU 的门控结构就是为了解决这个问题：哪些信息留下，哪些信息丢掉，不再完全交给普通循环更新。

## 17.9 LSTM 的门控直觉

LSTM 的核心是细胞状态 $c_t$。它像一条相对稳定的信息通道，门控结构决定信息如何流动。

遗忘门：

$$
f_t = \sigma(W_f[h_{t-1}, x_t] + b_f)
$$

输入门：

$$
i_t = \sigma(W_i[h_{t-1}, x_t] + b_i)
$$

候选记忆：

$$
\tilde{c}_t = tanh(W_c[h_{t-1}, x_t] + b_c)
$$

细胞状态更新：

$$
c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t
$$

输出门：

$$
o_t = \sigma(W_o[h_{t-1}, x_t] + b_o)
$$

隐藏状态：

$$
h_t = o_t \odot tanh(c_t)
$$

这里的 $\odot$ 表示逐元素相乘。门的输出在 `0` 到 `1` 之间，可以理解为保留比例。

## 17.10 词表示和序列输入

文本不能直接输入神经网络，需要先把 token 变成向量。

常见方式：

| 表示 | 特点 |
| --- | --- |
| one-hot | 稀疏、高维，不包含语义距离 |
| word embedding | 稠密、低维，能表示语义相似性 |
| contextual embedding | 同一个词在不同上下文中表示不同 |

Word2Vec 这类词向量给每个词一个固定向量。BERT、GPT 这类模型生成的是上下文相关表示，例如“苹果”在水果语境和公司语境中表示会不同。

## 17.11 Teacher Forcing

在序列生成任务中，训练时常用 Teacher Forcing。

意思是：预测第 $t$ 个 token 时，模型输入的前一个 token 使用真实答案，而不是模型自己上一步生成的结果。

这样训练更稳定，但也会带来训练和推理不一致的问题：

| 阶段 | 前一个 token 来自哪里 |
| --- | --- |
| 训练 | 真实序列 |
| 推理 | 模型自己生成 |

如果模型推理时前面生成错了，后续错误可能不断累积。

## 17.12 RNN、LSTM 和 Transformer 的取舍

| 模型 | 优点 | 局限 |
| --- | --- | --- |
| RNN | 结构简单，能处理变长序列 | 长期依赖差，难并行 |
| LSTM / GRU | 缓解长期依赖问题 | 仍按时间步串行计算 |
| Transformer | 并行能力强，长距离依赖更直接 | 标准注意力复杂度高 |

现在 NLP 主流是 Transformer，但 RNN/LSTM 在一些低延迟、小模型、时间序列场景中仍然有价值。
