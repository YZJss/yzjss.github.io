# 第 22 章 迁移学习与预训练模型

迁移学习的核心思想是：把一个任务或大数据集上学到的知识，迁移到另一个任务上。

## 22.1 为什么需要迁移学习

很多实际任务数据量不大，从零训练深度模型容易过拟合，也需要大量算力。

预训练模型已经在大规模数据上学到通用特征。把它迁移到新任务上，可以：

- 加速训练。
- 降低数据需求。
- 提升小数据集效果。
- 减少从零设计模型的成本。

## 22.2 特征提取和微调

使用预训练模型有两种常见方式：

| 方法 | 做法 | 适合场景 |
| --- | --- | --- |
| 特征提取 | 冻结预训练模型，只训练新分类器 | 数据少、任务相近 |
| 微调 | 继续训练部分或全部参数 | 数据较多、任务有差异 |

图像任务中，可以用 ImageNet 上预训练的 VGG、ResNet 提取特征，再训练 SVM 或新的分类层。这属于作为特征提取器的迁移学习。

## 22.3 NLP 中的预训练

文本任务里，预训练模型先在大规模语料上学习语言规律，再迁移到具体任务。

BERT 的预训练任务包括：

- 掩码语言模型。
- 下一句预测。

使用 BERT 做情感分析时，通常会保留 BERT 主体，在顶部加一个分类层，然后在情感分析数据上微调。

## 22.4 词嵌入

词嵌入把词映射到低维、稠密的向量空间，用来捕捉语义信息。

传统 one-hot 表示维度高且稀疏，词与词之间没有直接语义距离。词嵌入能让语义相近的词在向量空间里更接近。

常见词向量或嵌入模型包括 Word2Vec、BGE、Qwen embedding 等。


## 22.5 Embedding、Reranker 和生成模型

在大模型应用中，Embedding、Reranker 和生成模型经常一起出现，但作用不同。

| 类型 | 作用 | 例子 |
| --- | --- | --- |
| Embedding 模型 | 把文本变成向量，用于召回相似内容 | BGE-M3、Qwen embedding |
| Reranker 模型 | 对召回结果重新排序 | Qwen reranker |
| 生成模型 | 根据上下文生成答案 | GPT、Qwen、DeepSeek |

所以如果题目问“下列哪个不属于词向量模型”，Reranker 通常不是词向量模型，它负责重排序。

## 22.6 BERT 微调流程

以情感分析为例：

1. 加载预训练 BERT。
2. 在顶部接一个分类层。
3. 根据数据量决定冻结哪些层。
4. 用较小学习率微调。
5. 在验证集上选择最好的模型。

数据很少时，可以冻结大部分 BERT 层，只训练顶部分类层；数据更多或领域差异较大时，可以微调更多层。
