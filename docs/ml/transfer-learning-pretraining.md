# 第 26 章 迁移学习与预训练模型

迁移学习的核心思想是：把一个任务或大数据集上学到的知识，迁移到另一个任务上。

## 26.1 为什么需要迁移学习

很多实际任务数据量不大，从零训练深度模型容易过拟合，也需要大量算力。

预训练模型已经在大规模数据上学到通用特征。把它迁移到新任务上，可以：

- 加速训练。
- 降低数据需求。
- 提升小数据集效果。
- 减少从零设计模型的成本。

## 26.2 特征提取和微调

使用预训练模型有两种常见方式：

| 方法 | 做法 | 适合场景 |
| --- | --- | --- |
| 特征提取 | 冻结预训练模型，只训练新分类器 | 数据少、任务相近 |
| 微调 | 继续训练部分或全部参数 | 数据较多、任务有差异 |

图像任务中，可以用 ImageNet 上预训练的 VGG、ResNet 提取特征，再训练 SVM 或新的分类层。这属于作为特征提取器的迁移学习。

## 26.3 NLP 中的预训练

文本任务里，预训练模型先在大规模语料上学习语言规律，再迁移到具体任务。

BERT 的预训练任务包括：

- 掩码语言模型。
- 下一句预测。

使用 BERT 做情感分析时，通常会保留 BERT 主体，在顶部加一个分类层，然后在情感分析数据上微调。

## 26.4 词嵌入

词嵌入把词映射到低维、稠密的向量空间，用来捕捉语义信息。

传统 one-hot 表示维度高且稀疏，词与词之间没有直接语义距离。词嵌入能让语义相近的词在向量空间里更接近。

常见词向量或嵌入模型包括 Word2Vec、BGE、Qwen embedding 等。

## 26.5 大模型的后训练

大语言模型通常先进行大规模预训练，然后经过后训练提高对话、指令遵循和推理能力。

常见阶段：

| 阶段 | 作用 |
| --- | --- |
| 预训练 | 学习通用语言和知识 |
| SFT | 学习按照指令回答 |
| 偏好对齐 | 让回答更符合人类偏好 |
| 强化学习 | 进一步优化目标行为 |

后训练不是重新学全部知识，而是在已有模型能力上调整输出行为和任务表现。

## 26.6 温度参数

生成模型常用温度控制输出随机性。

| 温度 | 输出特点 |
| --- | --- |
| 较低 | 更稳定、更保守 |
| 较高 | 更随机、更有创造性 |

温度越高，模型越可能选择低概率 token，因此创造性更强，但稳定性和可控性可能下降。

## 26.7 AI Agent

AI Agent 和普通聊天助手的区别在于是否具备闭环执行能力。

普通助手主要回答问题；Agent 通常能拆解任务、调用工具、执行操作、观察结果，并根据反馈继续行动。

一个简单 Agent 流程：

```text
接收任务 -> 规划步骤 -> 调用工具 -> 观察结果 -> 调整计划 -> 完成任务
```

Agent 的关键不只是语言理解，而是能把理解转化成可执行行动，并根据结果闭环修正。

## 26.8 使用预训练模型的注意事项

预训练模型不是万能的。

需要注意：

- 训练数据和目标任务是否相近。
- 是否存在数据泄漏。
- 微调数据是否足够干净。
- 模型输出是否稳定可靠。
- 是否需要领域知识和人工审核。

迁移学习的正确用法是利用已有表示，而不是盲目相信预训练模型一定适合所有任务。

## 26.9 预训练为什么有效

预训练有效的核心原因是：很多任务共享底层表示。

图像任务中，边缘、纹理、形状等特征在不同数据集之间可以复用。文本任务中，词义、语法、上下文关系也可以在不同任务之间复用。

预训练模型先在大规模数据上学到通用表示，再用少量任务数据调整到具体任务。

这比从零训练更稳，尤其适合标注数据少、模型较大的场景。

## 26.10 冻结哪些层

迁移学习中，一个常见问题是冻结多少层。

| 数据量 | 任务相似度 | 建议 |
| --- | --- | --- |
| 少 | 高 | 冻结大部分层，只训练分类头 |
| 少 | 低 | 冻结底层，微调高层 |
| 多 | 高 | 微调高层或全模型 |
| 多 | 低 | 可以全模型微调，但要小学习率 |

底层特征更通用，高层特征更任务相关。图像模型中，底层边缘纹理较通用；高层类别语义更依赖原始训练任务。

## 26.11 微调时的学习率

微调预训练模型时，学习率通常比从零训练更小。

原因是预训练参数已经有较好表示，过大的学习率可能破坏已有知识，这有时叫灾难性遗忘。

常见做法：

- 新增分类头使用较大学习率。
- 预训练主体使用较小学习率。
- 先冻结主体训练分类头，再逐步解冻。

## 26.12 BERT 微调流程

以文本分类为例，BERT 微调通常是：

```text
文本 -> tokenizer -> token ids -> BERT -> [CLS] 表示 -> 分类层 -> 类别概率
```

`[CLS]` 位置的输出常被当作整句表示，用于分类任务。

微调时需要注意：

| 项 | 说明 |
| --- | --- |
| max length | 太短会截断信息，太长会增加计算 |
| batch size | 受显存限制 |
| learning rate | 通常较小 |
| warmup | 训练初期稳定更新 |
| weight decay | 缓解过拟合 |

## 26.13 Embedding、Reranker 和生成模型

大模型应用里经常出现几类模型：

| 类型 | 作用 |
| --- | --- |
| Embedding 模型 | 把文本转成向量，用于召回和相似度计算 |
| Reranker 模型 | 对候选结果重新排序 |
| 生成模型 | 根据上下文生成答案 |

在 RAG 系统中，通常先用 embedding 从知识库召回候选文档，再用 reranker 精排，最后把文档交给生成模型回答。

Embedding 模型不是直接生成文本，它输出的是向量。Reranker 也不是负责长文本生成，而是判断 query 和文档的相关性。

## 26.14 Agent 的能力边界

Agent 的关键是闭环执行，但闭环不等于绝对可靠。

一个 Agent 至少需要：

- 理解目标。
- 拆解任务。
- 选择工具。
- 执行操作。
- 观察结果。
- 根据反馈修正。

风险也很明显：

- 工具调用可能失败。
- 计划可能分解错。
- 外部环境可能变化。
- 模型可能误判结果是否成功。

所以重要任务中，Agent 需要日志、权限控制、人工确认和回滚机制。
