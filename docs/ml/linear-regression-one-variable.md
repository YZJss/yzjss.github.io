# 第 3 章 优化基础

这一章专门讲“模型是怎么被训练出来的”。后面的线性回归、逻辑回归、神经网络、Transformer，本质上都离不开同一个框架：定义损失函数，然后用优化算法让损失变小。

## 3.1 模型、参数和目标函数

模型负责把输入 $x$ 映射成预测值 $\hat{y}$。参数决定模型具体长什么样。目标函数，也叫代价函数或损失函数，用来衡量预测和真实标签之间的差距。

例如线性模型：

$$
h_{\theta}(x)=\theta^Tx
$$

如果用平方误差衡量预测偏差：

$$
J(\theta)
=
\frac{1}{2m}
\sum_{i=1}^{m}
\left(h_{\theta}(x^{(i)})-y^{(i)}\right)^2
$$

训练模型就是寻找一组参数 $\theta$，让 $J(\theta)$ 尽量小。

## 3.2 梯度下降

梯度下降的更新公式是：

$$
\theta_j
\leftarrow
\theta_j
-
\alpha
\frac{\partial J(\theta)}{\partial \theta_j}
$$

其中 $\alpha$ 是学习率，控制每次走多远。

直观理解：

| 部分 | 含义 |
| --- | --- |
| $\frac{\partial J}{\partial \theta_j}$ | 当前参数方向上损失变化速度 |
| 负号 | 往损失下降的方向走 |
| $\alpha$ | 每一步的步长 |

如果梯度为正，说明增大参数会让损失上升，因此应该减小参数；如果梯度为负，说明增大参数会让损失下降，因此应该增大参数。

## 3.3 学习率

学习率太小，训练会很慢；学习率太大，可能越过最优点，导致损失震荡甚至发散。

训练时常见现象：

| 现象 | 可能原因 |
| --- | --- |
| 损失下降很慢 | 学习率太小 |
| 损失上下震荡 | 学习率太大 |
| 损失变成异常值 | 学习率过大、梯度爆炸、数值不稳定 |
| 训练一开始就不稳定 | 初始化或学习率不合适 |

学习率是最重要的超参数之一。

## 3.4 批量梯度下降、随机梯度下降和小批量梯度下降

按照每次更新使用多少样本，可以分成三类：

| 方法 | 每次用多少数据 | 特点 |
| --- | --- | --- |
| 批量梯度下降 | 全部训练集 | 梯度稳定，但每步计算慢 |
| 随机梯度下降 | 1 个样本 | 更新快，但噪声大 |
| 小批量梯度下降 | 一小批样本 | 深度学习最常用，效率和稳定性折中 |

小批量梯度下降通常使用 `batch size` 控制每次训练的样本数量。batch 太小，梯度噪声大；batch 太大，显存压力大，也可能泛化变差。

## 3.5 动量

普通梯度下降每一步只看当前梯度，容易在狭长谷底来回震荡。动量会累积过去的更新方向。

直观上，动量像给参数更新加了惯性：如果连续几步方向一致，就加速；如果方向来回变化，就抵消震荡。

考试中常见说法是：动量可以在相关方向上加速学习，同时抑制震荡；它相当于对梯度做指数加权移动平均。

## 3.6 优化器和模型本身的区别

模型定义了预测形式，损失函数定义了什么叫“错”，优化器定义了如何根据错误更新参数。

| 名称 | 作用 |
| --- | --- |
| 模型 | 从输入得到预测 |
| 损失函数 | 衡量预测和真实值差距 |
| 反向传播 | 计算梯度 |
| 优化器 | 使用梯度更新参数 |

不要把反向传播和梯度下降混在一起。反向传播负责算梯度，梯度下降或 Adam 负责用梯度更新参数。

## 3.7 优化和泛化

优化好不等于泛化好。模型在训练集损失很低，可能只是记住了训练集噪声。泛化能力要看验证集和测试集表现。
