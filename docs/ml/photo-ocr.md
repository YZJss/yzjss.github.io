# 第 15 章 Transformer

Transformer 是现代深度学习中非常重要的结构，广泛用于大语言模型、机器翻译、文本分类、图像识别和多模态任务。

## 15.1 为什么需要注意力

RNN 按顺序处理序列，长距离依赖容易被逐步压缩。注意力机制允许模型直接比较序列中任意两个位置，从而更容易捕捉长距离关系。

自注意力的核心作用是：捕捉序列中不同元素之间的依赖关系。

## 15.2 Q、K、V

自注意力中，每个输入位置都会生成三个向量：

| 向量 | 含义 |
| --- | --- |
| Query | 当前我要找什么信息 |
| Key | 我能提供什么索引信息 |
| Value | 真正被汇总的内容 |

注意力权重由 Query 和 Key 的相似度决定，然后用这些权重加权 Value。

缩放点积注意力：

$$
Attention(Q,K,V)
=
softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

其中 $d_k$ 是 Key 的维度。除以 $\sqrt{d_k}$ 是为了避免点积过大导致 softmax 过于尖锐。

## 15.3 多头注意力

多头注意力会把表示拆成多个子空间，每个头学习不同关系。

好处是：模型可以从不同表示子空间关注不同位置的信息。

例如在一句话中，有的头可能关注主谓关系，有的头可能关注指代关系，有的头可能关注局部短语结构。

## 15.4 位置编码

自注意力本身不包含顺序信息。如果打乱输入顺序，注意力计算并不会天然知道哪个词在前、哪个词在后。

因此 Transformer 需要位置编码。

常见位置编码：

| 类型 | 说明 |
| --- | --- |
| 正弦余弦位置编码 | 固定函数生成 |
| 可学习位置编码 | 位置向量通过训练学习 |
| 相对位置编码 | 关注元素之间的相对距离 |

正弦和余弦位置编码的好处是可以帮助模型表达相对位置信息。

## 15.5 Encoder 和 Decoder

原始 Transformer 由 Encoder 和 Decoder 组成。

Encoder 读取输入序列，生成上下文表示。Decoder 根据已经生成的内容和 Encoder 输出逐步生成目标序列。

| 模块 | 常见任务 |
| --- | --- |
| Encoder | 文本分类、理解任务、BERT |
| Decoder | 文本生成、GPT |
| Encoder-Decoder | 翻译、摘要、T5 |

## 15.6 Masked Self-Attention

在生成任务中，模型预测当前位置时不能看到未来 token。

Masked Self-Attention 会屏蔽未来位置，保证第 $t$ 个位置只能关注 $1$ 到 $t$ 的内容。

这就是 Transformer 解码器中使用掩码自注意力的目的：防止模型在预测当前位置时看到未来信息。

## 15.7 前馈网络、残差连接和层归一化

Transformer 层通常包含：

```text
自注意力 -> 残差连接 -> LayerNorm -> 前馈网络 -> 残差连接 -> LayerNorm
```

前馈网络对每个位置独立作用，负责进一步变换表示。残差连接帮助深层网络训练，LayerNorm 稳定训练过程。

## 15.8 复杂度问题

标准自注意力需要计算所有位置两两关系，复杂度是 $O(N^2)$。

当序列很长时，$O(N^2)$ 会成为主要瓶颈。

一些改进方向：

| 方法 | 思路 |
| --- | --- |
| 稀疏注意力 | 只关注部分位置 |
| 局部注意力 | 只看附近窗口 |
| Reformer | 使用 LSH 降低复杂度 |
| Longformer | 结合局部和全局注意力 |

## 15.9 BERT 和 GPT

BERT 偏理解，GPT 偏生成。

| 模型 | 结构 | 训练目标 |
| --- | --- | --- |
| BERT | Encoder | 掩码语言模型等 |
| GPT | Decoder | 预测下一个 token |

BERT 做情感分析时，通常加载预训练模型，然后在任务数据上微调顶部分类层或整个模型。

## 15.10 注意力计算的完整过程

设输入序列表示为矩阵 $X$，每一行对应一个 token 的向量表示。

通过三个线性变换得到：

$$
Q = XW_Q
$$

$$
K = XW_K
$$

$$
V = XW_V
$$

然后计算注意力分数：

$$
S = \frac{QK^T}{\sqrt{d_k}}
$$

$S_{ij}$ 表示第 $i$ 个位置对第 $j$ 个位置的关注程度。经过 Softmax 后，每一行变成权重分布：

$$
A = softmax(S)
$$

最后加权求和：

$$
O = AV
$$

所以注意力不是“随便关注”，而是一个可微的加权汇总过程，权重由 Query 和 Key 的相似度决定。

## 15.11 为什么要多头

单头注意力只有一个相似度空间，表达能力有限。多头注意力把表示分成多个子空间，分别做注意力，再拼接起来。

可以理解为不同头有不同关注角度：

- 有的头关注相邻词。
- 有的头关注句法依赖。
- 有的头关注长距离指代。
- 有的头关注特殊标点或分隔符。

多头注意力不是简单重复计算，而是让模型在多个投影空间中学习不同关系。

## 15.12 残差连接的位置

Transformer 每个子层通常都有残差连接：

$$
x_{out} = x + Sublayer(x)
$$

然后再做归一化。不同实现有 Post-LN 和 Pre-LN 两种常见结构。

Post-LN：

```text
x -> Sublayer -> Add -> LayerNorm
```

Pre-LN：

```text
x -> LayerNorm -> Sublayer -> Add
```

Pre-LN 在训练很深的 Transformer 时通常更稳定，因为梯度路径更顺。

## 15.13 前馈网络为什么需要

自注意力负责混合不同位置的信息，但它本身主要是在 token 之间做加权汇总。

前馈网络负责对每个位置的表示做非线性变换：

$$
FFN(x) = W_2 \phi(W_1x + b_1) + b_2
$$

其中 $\phi$ 可以是 ReLU 或 GELU。

可以粗略理解：

- 注意力层负责“和谁交换信息”。
- 前馈网络负责“交换后怎么加工信息”。

## 15.14 Encoder-only、Decoder-only 和 Encoder-Decoder

三种结构适合不同任务：

| 结构 | 注意力方式 | 典型模型 | 适合任务 |
| --- | --- | --- | --- |
| Encoder-only | 双向注意力 | BERT | 理解、分类、抽取 |
| Decoder-only | 因果掩码注意力 | GPT | 生成、对话 |
| Encoder-Decoder | 编码输入，解码输出 | T5、原始 Transformer | 翻译、摘要 |

BERT 能同时看左右上下文，所以适合理解任务。GPT 只能看当前位置之前的 token，所以适合自回归生成。

## 15.15 长序列瓶颈

标准注意力的瓶颈来自注意力矩阵。

如果序列长度是 $N$，注意力矩阵大小是：

$$
N \times N
$$

长度翻倍，注意力计算和显存开销大约变成四倍。这就是长上下文模型需要改进注意力机制的原因。

常见思路包括稀疏注意力、滑动窗口注意力、低秩近似、哈希注意力、状态空间模型等。不同方法是在计算效率、表达能力和实现复杂度之间权衡。

## 15.16 大语言模型

大语言模型通常指参数规模大、训练数据规模大、通用能力强的语言模型。它们一般不是为单一任务从零训练，而是先通过大规模预训练学习通用语言表示，再通过后训练或微调适配具体任务。

大语言模型的基本训练目标通常是根据上下文预测下一个 token：

$$
P(x_t \mid x_1, x_2, \cdots, x_{t-1})
$$

这个目标看起来简单，但在海量文本上训练后，模型会学到语法、知识、推理模式和任务格式。

## 15.17 预训练和后训练

大模型训练通常分为两个阶段：

| 阶段 | 作用 |
| --- | --- |
| 预训练 | 学习通用语言、知识和表示 |
| 后训练 | 学会按照指令、偏好和任务要求输出 |

后训练不是重新学习全部知识，而是在已有能力基础上调整行为。常见后训练包括 SFT、偏好对齐和强化学习。

## 15.18 SFT

SFT 是有监督微调。训练数据通常是人工整理的指令和回答，让模型学会按照指令输出。

例如：

```text
用户：解释什么是正则化
助手：正则化是……
```

SFT 可以明显提升模型的指令遵循能力，但如果只做 SFT，模型可能还不够符合人类偏好，也不一定具备强推理能力。

## 15.19 强化学习后训练

强化学习后训练使用奖励信号优化模型输出。奖励可以来自人类偏好，也可以来自规则或自动验证器。

如果任务有明确可验证结果，例如数学、代码、逻辑题，规则奖励可以让模型通过试错学会更好的推理策略。

这也是一些模型在不依赖大量人类示例的情况下，通过强化学习提升推理能力的原因。

## 15.20 温度参数

生成时，模型会给每个候选 token 一个概率。温度参数控制这个概率分布的平滑程度。

| 温度 | 输出特点 |
| --- | --- |
| 低温度 | 更稳定、更保守、更确定 |
| 高温度 | 更随机、更有创造性、风险更高 |

温度越高，低概率 token 被采样到的机会越大，因此创造性更强，但事实性和可控性可能下降。

## 15.21 MoE

MoE 是 Mixture of Experts，混合专家模型。它包含多个专家网络，但每个 token 通常只激活其中一部分专家。

MoE 的优势是：在相同推理成本下，可以拥有更大的总参数量，从而提升模型能力。

但 MoE 也有代价，例如路由策略、负载均衡、通信开销和部署复杂度。

## 15.22 推理能力

推理能力不只是生成更长答案，而是能分解问题、保持中间状态、检查约束并逐步求解。

影响推理能力的因素包括：

- 预训练数据中是否包含高质量推理样本。
- 后训练是否鼓励逐步求解。
- 奖励信号是否能区分正确和错误推理。
- 推理时是否允许更长计算过程。

## 15.23 幻觉

幻觉指模型生成看似合理但实际上错误的内容。它可能来自训练数据错误、模型记忆不完整、上下文不足或生成目标本身只追求“像真的”。

常见缓解方式：

- RAG 检索外部资料。
- 工具调用，把搜索、计算、数据库查询交给外部系统。
- 要求引用来源。
- 对关键场景加入人工审核。
- 限制模型在不知道时必须说明不确定。

幻觉不能完全靠提示词消除，工程上要靠系统设计降低风险。

## 15.24 大模型应用中的三类模型

大模型应用里常见三类模型：

| 类型 | 作用 |
| --- | --- |
| Embedding 模型 | 把文本转成向量，用于召回 |
| Reranker 模型 | 对候选文档重新排序 |
| 生成模型 | 生成最终回答 |

Embedding 和 Reranker 的细节放在迁移学习章节，这里只记住它们通常服务于 RAG 流程。
