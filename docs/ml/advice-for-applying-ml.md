# 第 6 章 模型评估与泛化

训练模型时，最容易犯的错误是只盯着训练集表现。训练误差低只能说明模型在见过的数据上表现好，却不能保证它对新样本可靠。机器学习真正关心的是泛化能力，也就是模型在未见过的数据上的表现。

本章讨论如何评估模型、如何判断欠拟合和过拟合、如何选择模型复杂度，以及为什么数据泄漏会让评估结果失真。

## 6.1 训练集、验证集和测试集

为了估计泛化能力，通常把数据分成三部分。

训练集用于拟合参数。线性回归中的 $\theta$、神经网络中的权重，都是从训练集中学到的。

验证集用于选择模型和超参数。例如选择多项式阶数、正则化强度、学习率、网络层数时，应该看验证集表现。

测试集只用于最终评估。它应该尽量模拟模型上线后会遇到的新数据。如果在调参过程中反复查看测试集表现，测试集就不再是“未见过的数据”，评估会偏乐观。

可以把三者理解为：

| 数据集 | 作用 |
| --- | --- |
| 训练集 | 学参数 |
| 验证集 | 选模型和超参数 |
| 测试集 | 最终估计泛化能力 |

## 6.2 评估一个假设

模型训练完成后，不能只报告训练误差。一个高阶多项式模型可能把训练集拟合得很好，但对新数据预测很差。

对于回归任务，常用均方误差或平均绝对误差。对于分类任务，常用准确率、精确率、召回率、F1 等指标。指标选择取决于任务目标。例如垃圾邮件识别中，误杀正常邮件的代价很高，就不能只看准确率。

评估指标应该在建模前就确定，而不是模型训练后再挑一个看起来最好的指标。

## 6.3 模型选择

模型选择是选择假设空间的过程。以多项式回归为例，可以选择一次、二次、三次或更高阶模型。

如果只看训练误差，高阶模型几乎总是更好，因为它表达能力更强。但表达能力更强不代表泛化更好。因此应该用验证集选择模型复杂度。

典型流程：

1. 在训练集上训练多个候选模型。
2. 在验证集上比较它们的误差。
3. 选择验证集表现最好的模型。
4. 最后只用测试集评估一次。

这能避免把测试集也变成调参工具。

## 6.4 欠拟合和过拟合

欠拟合指模型太简单，训练集都拟合不好。过拟合指模型太复杂，把训练集中的噪声也学进去了。

两者可以通过训练误差和验证误差判断：

| 情况 | 训练误差 | 验证误差 |
| --- | --- | --- |
| 欠拟合 | 高 | 高 |
| 过拟合 | 低 | 高 |
| 合适拟合 | 低 | 低 |

如果训练集和验证集误差都很高，说明模型连训练数据规律都没有学好，通常是高偏差问题。如果训练误差很低而验证误差很高，说明模型对训练集记得太细，通常是高方差问题。

## 6.5 偏差和方差

偏差可以理解为模型假设和真实规律之间的系统性差距。方差可以理解为模型对训练数据扰动的敏感程度。

高偏差模型太简单。例如用直线拟合明显弯曲的数据。

高方差模型太敏感。例如高阶多项式穿过几乎所有训练点，但在点与点之间剧烈摆动。

常见处理方式：

| 问题 | 可能做法 |
| --- | --- |
| 高偏差 | 增加特征、提高模型复杂度、训练更久 |
| 高方差 | 增加数据、加强正则化、降低模型复杂度 |

## 6.6 学习曲线

学习曲线观察训练集大小变化时训练误差和验证误差如何变化。

如果训练误差和验证误差都很高，而且两者接近，通常是高偏差。此时增加更多数据帮助有限，更应该考虑换更复杂的模型或增加有效特征。

如果训练误差低、验证误差高，且两者差距大，通常是高方差。此时增加训练数据往往有帮助，正则化也可能有效。

学习曲线的意义在于帮助决定下一步该做什么，而不是盲目尝试。

## 6.7 正则化强度和泛化

正则化强度也会影响偏差和方差。

如果 $\lambda$ 太大，参数被压得过小，模型可能欠拟合。此时训练误差和验证误差都会偏高。

如果 $\lambda$ 太小，模型约束不足，可能过拟合。此时训练误差低，验证误差高。

因此正则化参数应该通过验证集选择，而不是凭感觉固定。

## 6.8 超参数

超参数是在训练开始前设置的参数，不是模型从数据中自动学出来的参数。

常见超参数包括：

- 学习率。
- 正则化强度。
- 多项式阶数。
- 隐藏层数量。
- 隐藏单元数量。
- batch size。
- dropout rate。

超参数选择属于模型选择的一部分。合理做法是用训练集训练，用验证集选择，最后用测试集评估。

## 6.9 数据泄漏

数据泄漏指训练过程中使用了本不该知道的信息。它会让验证或测试结果看起来很好，但上线后表现下降。

常见例子包括：

- 在划分训练集和测试集之前，先用全量数据做标准化。
- 特征中包含未来信息。
- 同一个用户或同一张图片的增强版本同时出现在训练集和测试集。
- 多次根据测试集结果调整模型。

防止数据泄漏的原则是：任何从数据中估计出来的处理步骤，都应该只在训练集上拟合，再应用到验证集和测试集。
