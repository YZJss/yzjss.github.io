第 11 章 决策树 ​

随机森林是一种集成学习方法。它不是训练一棵决策树，而是训练多棵决策树，再把这些树的结果综合起来。

随机森林常用于表格数据，优点是效果稳定、对特征缩放不敏感、能处理非线性关系，也不容易像单棵决策树那样过拟合。

11.1 决策树 ​

决策树通过一系列判断条件把样本划分到不同叶子节点。

例如：

text
年龄 < 30 ?
  是 -> 收入 < 8000 ?
  否 -> 工作年限 < 5 ?

每个内部节点是一个划分规则，每个叶子节点给出预测结果。

决策树可以用于分类，也可以用于回归。

任务叶子节点输出
分类类别或类别概率
回归数值预测

11.2 划分标准 ​

决策树训练时要选择特征和阈值，让划分后的数据更“纯”。

分类树常用指标：

指标含义
信息增益划分后熵减少多少
增益率对信息增益做修正
基尼指数衡量类别混杂程度

回归树常用平方误差来选择划分，使划分后每个区域内的目标值更接近。

11.3 单棵决策树的问题 ​

决策树的优点是直观、可解释、不需要特征缩放。

但单棵决策树容易过拟合。树越深，就越可能把训练集中的噪声也记住。

常见控制方法：

  • 限制最大深度。
  • 限制叶子节点最小样本数。
  • 剪枝。
  • 限制每次划分可用特征。

随机森林就是在“多棵不完全相同的树”基础上缓解过拟合。

11.4 Bagging ​

Bagging 是 Bootstrap Aggregating 的缩写。

它的基本思想：

  1. 从训练集中有放回地抽样，得到多个不同子数据集。
  2. 在每个子数据集上训练一个模型。
  3. 把多个模型的预测结果综合起来。

分类任务通常投票：

text
多数树预测为哪一类，最终就预测哪一类

回归任务通常取平均：

text
多棵树预测值的平均值作为最终输出

Bagging 可以降低方差，让模型结果更稳定。

11.5 随机森林 ​

随机森林可以看作 Bagging + 决策树 + 特征随机选择。

它的随机性主要来自两处：

随机性作用
样本随机每棵树使用 bootstrap 抽样得到的数据
特征随机每个节点划分时只从部分特征中选择

如果每棵树都很相似，集成效果有限。样本随机和特征随机会让树之间产生差异，从而降低整体方差。

11.6 训练和预测 ​

训练过程：

  1. 生成多个 bootstrap 数据集。
  2. 在每个数据集上训练一棵决策树。
  3. 每个节点划分时，只随机选择一部分特征参与比较。
  4. 得到多棵树组成的森林。

预测过程：

任务随机森林输出
分类多数投票
回归平均预测值

11.7 随机森林的优缺点 ​

优点：

  • 能处理非线性关系。
  • 不需要特征缩放。
  • 对异常值相对鲁棒。
  • 比单棵决策树更不容易过拟合。
  • 可以估计特征重要性。

缺点：

  • 可解释性不如单棵决策树。
  • 模型体积更大。
  • 预测速度比单棵树慢。
  • 对高维稀疏文本特征不一定是最佳选择。

11.8 和其他模型的区别 ​

模型特点
逻辑回归线性分类模型，简单可解释
SVM关注间隔，可配合核函数处理非线性
决策树规则划分，可解释但容易过拟合
随机森林多棵树集成，稳定性更好

随机森林属于 Bagging 思想，重点是降低方差。后续如果学习 GBDT、XGBoost、LightGBM，它们属于 Boosting 思想，重点是逐步修正前面模型的错误。
