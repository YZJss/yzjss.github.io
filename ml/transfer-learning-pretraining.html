第 21 章 迁移学习 ​

深度模型通常需要大量数据和算力。如果每个任务都从零开始训练，成本很高，也容易过拟合。迁移学习的思想是：先在大规模数据上学习通用表示，再把这些表示迁移到新的任务中。

预训练模型之所以有效，是因为很多任务共享底层结构。图像任务共享边缘、纹理、形状等视觉特征；文本任务共享词义、语法和上下文关系。预训练模型先学到这些通用表示，再用较少的任务数据进行调整。

21.1 特征提取 ​

最简单的迁移方式是把预训练模型当作特征提取器。

例如图像分类中，可以使用在 ImageNet 上预训练的 VGG 或 ResNet。去掉最后的分类层，把中间层输出作为图像特征，再训练一个新的分类器，例如 SVM 或线性分类层。

这种方法适合数据量较小、目标任务和预训练任务相近的场景。因为大部分网络参数被冻结，训练成本低，也不容易过拟合。

流程可以写成：

text
输入图像 -> 预训练 CNN -> 特征向量 -> 新分类器

21.2 微调 ​

微调会在新任务上继续训练预训练模型的部分或全部参数。

如果新任务数据较少，通常只训练顶部几层，冻结底层特征提取层。如果新任务数据较多，或者和预训练任务差异较大，可以解冻更多层甚至全模型微调。

经验上，微调时学习率通常要比从零训练小。原因是预训练参数已经包含有用表示，过大的学习率可能破坏这些表示。

可以按下面原则选择：

数据量任务相似度做法
少高冻结主体，只训练分类头
少低冻结底层，微调高层
多高微调高层或全模型
多低全模型微调，但使用较小学习率

21.3 视觉预训练 ​

视觉任务中，ImageNet 预训练长期是常见做法。浅层卷积核通常学习边缘、颜色和纹理，这些特征在许多视觉任务中都能复用。

例如医学图像数据量较少时，可以先使用 ImageNet 预训练模型，再在医学数据上微调。虽然自然图像和医学图像不同，但底层视觉特征仍可能提供较好的初始化。

使用预训练模型并不意味着一定更好。如果目标数据和预训练数据差异太大，迁移效果可能有限，甚至出现负迁移。

21.4 NLP 中的预训练 ​

自然语言处理中，预训练模型先在大规模语料上学习语言规律，再迁移到分类、问答、摘要、信息抽取等任务。

BERT 是典型的预训练语言模型。它的预训练任务包括：

  • 掩码语言模型。
  • 下一句预测。

掩码语言模型会随机遮住部分 token，让模型根据上下文预测被遮住的词。这样模型能学习双向上下文表示。

21.5 BERT 微调 ​

使用 BERT 做情感分析时，通常在 BERT 顶部加一个分类层。输入句子经过 BERT 得到表示，再由分类层输出情感类别。

典型流程：

  1. 加载预训练 BERT。
  2. 添加任务相关输出层。
  3. 用任务数据训练分类层和部分 BERT 参数。
  4. 在验证集上选择最好的学习率和训练轮数。

如果数据很少，可以冻结大部分层，只训练分类头；如果数据较多，可以微调整个模型。

21.6 词嵌入 ​

词嵌入把离散词映射到低维稠密向量空间。

one-hot 表示虽然简单，但维度高且稀疏，并且不能表达词之间的相似性。词嵌入则可以让语义相近的词在向量空间中更接近。

例如“国王”和“王后”、“北京”和“中国”之间可能在向量空间中形成有意义的关系。

Word2Vec 是经典词向量模型。现代系统中还会使用 BGE、Qwen embedding 等模型生成句子或文档级向量。

21.7 Embedding、Reranker 和生成模型 ​

在 RAG 或搜索系统中，Embedding、Reranker 和生成模型分工不同。

Embedding 模型负责把查询和文档编码成向量，用于快速召回候选文档。

Reranker 模型负责对候选文档重新排序。它通常比 embedding 召回更精细，但计算成本更高。

生成模型负责根据最终上下文生成答案。

所以它们的关系通常是：

text
Embedding 召回 -> Reranker 精排 -> 生成模型回答

如果题目问某个模型是否属于词向量模型，要看它的主要功能。如果是 reranker，它通常不是词向量模型，而是排序模型。

21.8 使用预训练模型的风险 ​

预训练模型不是万能的。使用时要注意：

  • 预训练数据和目标任务是否相近。
  • 微调数据是否足够干净。
  • 是否存在数据泄漏。
  • 模型是否学到了不适合当前场景的偏差。
  • 输出是否需要人工审核。

迁移学习的正确用法是利用已有表示，而不是盲目相信预训练模型一定适合所有任务。
